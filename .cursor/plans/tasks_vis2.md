# Jamming Bot — Новые визуализации (Data-driven Generative Art)

## Цель
Добавить три новые художественные визуализации на основе существующих NLP-данных (сущности, фразы, векторы, sentiment, частоты, co-occurrence и т.д.).  
Все новые визуализации должны сохранять постапокалиптическую, glitch, минималистичную и «цифровой распад» эстетику проекта.

## Новые теги для реализации

### 4. 2D Constellation (constellation)
**Описание:**  
2D-звёздное небо (или космическая карта), где каждое слово/сущность — звезда.  
Яркость и размер звезды зависят от частоты.  
Линии-связи (созвездия) рисуются между словами с высоким семантическим сходством или co-occurrence.  
Анимация: лёгкое мерцание, медленное парение звёзд, периодическое появление новых связей.

**Технические требования:**
- Использовать Canvas 2D или SVG + D3.js / Three.js (2D mode)
- Рассчитывать позиции через force-directed layout или простую круговую/спиральную раскладку
- Связи: на основе cosine similarity векторов spaCy или dependency parsing
- Добавить лёгкий glitch-эффект на линии
- Новый маршрут: `/tags/constellation`

**Статус:** ☐ Не начато

### 8. Vector Field (vectorfield)
**Описание:**  
Абстрактное поле векторов (как магнитные силовые линии или поток жидкости).  
Каждый вектор/кривая отражает направление и силу семантического вектора слова.  
Цвет и толщина зависят от категории (сущность, sentiment, тема).  
Анимация: непрерывное течение, лёгкие завихрения, появление/исчезновение потоков при новом «джеме».

**Технические требования:**
- WebGL / Three.js + Shader (рекомендуется) или particles с velocity
- Использовать word embeddings (spaCy vectors) для направления векторов
- Можно применить noise (Perlin или simplex) для органичности
- Постапокалиптический вид: частицы с лёгким «шумом» и цифровым распадом
- Новый маршрут: `/tags/vectorfield`

**Статус:** ☐ Не начато

### 10. Chaos Attractor (chaos-attractor)
**Описание:**  
Динамическая визуализация странного аттрактора (вдохновлено аттрактором Лоренца или Рössler).  
Параметры аттрактора управляются статистикой текущего текста:  
- энтропия текста → сила хаоса  
- variance частот → масштаб  
- доминирующие темы → цветовые сдвиги  
Траектории частиц оставляют следы, создавая сложный, гипнотический паттерн, который постоянно эволюционирует.

**Технические требования:**
- Three.js + particle system или GPU-шейдеры
- Реализовать простую систему дифференциальных уравнений (Lorenz attractor как база)
- Маппинг данных: entropy, topic distribution, sentiment variance
- Анимация в реальном времени с возможностью «встряхивания» при новом джеме
- Новый маршрут: `/tags/chaos-attractor`

**Статус:** ☐ Не начато

## Общие требования ко всем трём визуализациям

1. Поддержка мобильных устройств и десктопа (responsive)
2. Единый стиль: тёмная палитра (#0a0a0a, неоновые акценты, glitch, scanlines, лёгкий noise)
3. Использовать существующий пайплайн данных (JSON от бэкенда с NLP-результатами)
4. Добавить в систему тегов (`/tags/`) и на главную страницу возможность выбора
5. Оптимизация производительности (60 fps даже с большим количеством частиц)
6. Возможность «заморозки» текущей визуализации и экспорта как изображение
7. Код должен быть модульным и легко расширяемым (отдельный файл/компонент для каждой визуализации)

## Порядок реализации (рекомендуется)
1. **constellation** — самый простой и быстрый для прототипирования
2. **vectorfield** — средняя сложность
3. **chaos-attractor** — наиболее вычислительно тяжёлый и красивый
