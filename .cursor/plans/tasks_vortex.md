# Jamming Bot — Задача: Реализация 4. Sentiment Vortex

## Название визуализации
**Sentiment Vortex** (вихрь / воронка)

## Описание
Создать гипнотическую визуализацию в виде вихря/воронки, где:
- Частицы движутся по спирали к центру или от центра
- **Цвет** частиц зависит от эмоциональной окраски текста (polarity)
- **Направление** движения и скорость зависят от polarity и subjectivity
- **Размер** и **яркость** частиц зависят от интенсивности sentiment

Это должна быть одна из самых атмосферных и эмоциональных визуализаций бота — «эмоциональный шторм» последнего цифрового существа.

## Требования к визуализации

### Поведение частиц:
- Частицы движутся по спиральной траектории (вихрь)
- При **положительном** polarity (positive) — движение преимущественно **от центра** (расширение)
- При **отрицательном** polarity (negative) — движение **к центру** (засасывание)
- При нейтральном — мягкое круговое движение
- **Subjectivity** (субъективность) влияет на хаотичность движения: чем выше — тем больше случайных отклонений и турбулентности

### Цветовая схема:
- Positive sentiment → тёплые неоновые цвета (розовый, оранжевый, золотой, ярко-зелёный)
- Negative sentiment → холодные/тёмные цвета (фиолетовый, синий, малиново-красный, чёрный с красным свечением)
- Нейтральный → серо-голубые, приглушённые тона
- Интенсивность цвета зависит от абсолютного значения polarity

### Дополнительные эффекты (для атмосферы):
- Лёгкий glitch-эффект на частицах при сильных эмоциональных всплесках
- Тонкие «хвосты» (trails) за частицами
- Лёгкий radial blur или glow в центре воронки
- Постепенное появление/исчезновение частиц при обновлении данных
- Слабый noise / digital distortion по всему полю

## Технические требования

- **Технологии:** Three.js (рекомендуется) или Canvas 2D + particles. Предпочтительно Three.js для красивых эффектов.
- **Данные из бэкенда:** 
  - Массив фраз/слов с полями: `polarity`, `subjectivity`, `text`
  - Общая статистика sentiment (средний polarity, процент positive/negative/neutral)
- Новый маршрут: `/tags/sentiment-vortex`
- Добавить тег в список на странице `/tags/`
- Визуализация должна работать в реальном времени и обновляться при новом "джеме" данных

## Этапы реализации

1. **Создать новый компонент** `SentimentVortex.vue` или `sentiment-vortex.js`
2. **Получить данные** от существующего API эндпоинта (NLP-результаты с sentiment)
3. **Реализовать particle system** с индивидуальными параметрами для каждой частицы
4. **Маппинг sentiment → визуальные параметры** (цвет, скорость, направление, турбулентность)
5. **Добавить спиральную физику** движения
6. **Настроить постапокалиптическую эстетику** (тёмный фон, неон, glitch)
7. **Добавить UI-элемент** для выбора этой визуализации
8. **Оптимизировать производительность** (цель — 60 fps)
